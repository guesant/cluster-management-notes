> **Para quem é:** operadores com OpenBao rodando que querem eliminar unseal manual via KMS externo.
> **Pré-requisito:** [OpenBao já instalado](install-openbao/) + acesso ao KMS (AWS/GCP/Azure).

Auto-unseal permite que OpenBao se desencripte automaticamente na inicialização usando um serviço KMS externo. Este guia cobre setup com **AWS KMS** (adaptável a GCP/Azure).

## Passo 2: Criar IAM role e credenciais

OpenBao precisa de permissões para acessar a chave KMS. Criar uma role:

<ScriptHelper
  runWhere="máquina com `aws` CLI configurado"
  script={openbaoAwsIamSetupScript}
  fields={[
    { var: 'KMS_KEY_ARN', label: 'ARN da chave KMS (do passo anterior)', defaultValue: 'arn:aws:kms:us-east-1:123456789:key/abc-def-ghi' },
    { var: 'IAM_ROLE_NAME', label: 'Nome da role IAM', defaultValue: 'openbao-unseal' },
  ]}
/>

**Se executar em Kubernetes (K3s):** usar IRSA (IAM Roles for Service Accounts) ou assumir role via `aws sts assume-role` dentro do pod.

## Passo 3: Atualizar configuração de OpenBao

Editar o arquivo de configuração de OpenBao (`/etc/openbao/openbao.hcl` ou via Helm):

<FileWriter path="openbao.hcl" lang="hcl">
{`ui = true

storage "postgresql" {
  connection_string = "postgres://bao:password@postgres-host:5432/openbao"
}

seal "awskms" {
  region     = "us-east-1"
  kms_key_id = "arn:aws:kms:us-east-1:123456789:key/abc-def-ghi"
}

listener "tcp" {
  address       = "0.0.0.0:8200"
  tls_cert_file = "/etc/openbao/certs/bao.crt"
  tls_key_file  = "/etc/openbao/certs/bao.key"
}`}
</FileWriter>

**Para GCP (Google Cloud KMS):**

<FileWriter path="openbao-gcp.hcl" lang="hcl">
{`seal "gcpkms" {
  project = "my-project"
  region  = "us-central1"
  key_ring = "openbao-ring"
  crypto_key = "openbao-key"
}`}
</FileWriter>

**Para Azure:**

<FileWriter path="openbao-azure.hcl" lang="hcl">
{`seal "azurekeyvault" {
  tenant_id      = "your-tenant-id"
  client_id      = "your-client-id"
  client_secret  = "your-client-secret"
  vault_name     = "my-vault"
  key_name       = "openbao-unseal-key"
}`}
</FileWriter>

## Passo 4: Reiniciar OpenBao

```
# Se usando systemd
sudo systemctl restart openbao

# Se usando Docker
docker restart openbao

# Se usando Kubernetes
kubectl rollout restart -n openbao deployment/openbao
```

Aguardar ~10 segundos. Verificar que OpenBao iniciou sem intervenção:

```
bao status
```

Esperado:
```
Key                      Value
---                      -----
Seal Type                awskms
Initialized              true
Sealed                   false  ← auto-unsealed!
Standby                  false
Replication              primary
```

## Passo 5: Testar failover de unseal

Para confirmar que auto-unseal funciona:

```
# 1. Derrubar OpenBao (simular crash)
sudo systemctl stop openbao

# 2. Aguardar 5 seg
sleep 5

# 3. Reiniciar
sudo systemctl start openbao

# 4. Verificar que desencriptou automaticamente
bao status
# Esperado: Sealed = false (sem intervenção)
```

## Passo 6: Remover chaves de unseal antigas

Se OpenBao foi anteriormente configurado com Shamir keys, você pode remover:

```
bao operator key-status
# Se ainda tem Shamir keys, você pode remover (requer unseal ativo primeiro):
# (Não há comando direto; elas ficam obsoletas uma vez que auto-unseal está ativo)
```

## Problemas comuns

### "permission denied" ao conectar ao KMS

**Causa:** credenciais IAM não têm permissão.

**Solução:**

```
aws iam get-role-policy --role-name openbao-unseal --policy-name allow-kms-unseal

# Se em EC2, verificar que instância tem a role
aws ec2 describe-instances --instance-ids i-xxx | grep IamInstanceProfile

# Se em K3s com IRSA, verificar anotação do pod
kubectl describe pod -n openbao openbao-0 | grep iam
```

### "Seal Type: shamir" (não está usando KMS)

**Causa:** configuração não foi recarregada.

**Solução:**

```
cat /etc/openbao/openbao.hcl | grep -A 5 seal

# Se falta "seal" block, adicionar
# Reiniciar OpenBao completamente (não restart, stop+start)
sudo systemctl stop openbao
sleep 5
sudo systemctl start openbao
```

### "KMS key not found"

**Causa:** ARN da chave está incorreto ou chave foi deletada.

**Solução:**

```
aws kms list-keys --region us-east-1

# Verificar detalhes de uma chave
aws kms describe-key --key-id abc-def-ghi --region us-east-1
```
