Esta etapa produz um cluster funcional de nó único. Em uma topologia multinó, ela também inicializa o primeiro manager e o datastore etcd ao qual os demais servidores e agentes serão conectados.

## Montar o comando de instalação

Preencha os campos abaixo para montar o comando de instalação do K3s para o papel deste nó
(primeiro servidor, servidor adicional ou agente). O comando gerado usa o instalador oficial
`get.k3s.io` diretamente — é uma alternativa mais simples ao bloco com arquivo de configuração
persistente usado no restante desta página, útil para testes rápidos ou nós adicionais. Nenhum
valor preenchido sai do seu navegador.

## Instalação com arquivo de configuração persistente

Para o primeiro servidor, o roteiro recomendado por este guia grava as opções em
`/etc/rancher/k3s/config.yaml` em vez de passá-las só na linha de comando — isso facilita
reinstalações, auditoria e integração com backups do host.

Execute em um host novo. Por padrão, cada valor é perguntado no terminal via `read` — pressione
Enter no prompt do token para gerar um valor aleatório; o script exigirá a confirmação de que o
valor foi guardado antes de continuar. Cada campo pode também ser embutido no script gerado ou
criptografado com `age`, e o corpo do script pode ser editado antes de copiar.

> **Executar em:** host que será o primeiro nó manager, como `root`.
